# 🌱 Smartwater - 基于 HomeAssistant + LLM 的智能喷灌联动系统

**AgriBrain** 将原本封闭的商业智能喷灌设备接入 HomeAssistant 生态，并引入大语言模型（LLM）作为决策大脑，实现 **跨设备联动、多源农业数据融合** 与 **自然语言交互控制** 的下一代精准灌溉系统。

> 本项目为黑客松大赛概念验证原型，重点展示 **开放生态集成** 与 **LLM驱动决策** 的技术路径。

---

## 📌 背景与问题

市面上现有的智能喷灌系统（如某品牌的 `SmartIrrigator Pro`）已经拥有：

- ☀️ 太阳能供电 + 4G 远程遥控开关
- 📱 手机端控制浇水，定时任务
- 🔒 **但：系统完全闭源，无法与气象站、土壤传感器、智能家居联动**
- 🧠 **不具备基于多源数据的智能决策能力**

农户如果同时拥有土壤湿度计、小型气象站、虫情灯等设备，它们只能各自为政，无法形成闭环的自动化管理。

**AgriBrain 的目标**：  
将这种商业设备“撬开一条缝”，通过 HomeAssistant 的统一平台，让 LLM 能够像人类管家一样阅读传感器数值、天气预报，甚至接收你的自然语言指令，最终精准地拧开或关闭喷灌开关。

## ✨ 核心特性

- 🔌 **商业设备接入 HA**：通过自定义集成，将闭源设备的云 API 封装成 HomeAssistant 的 `switch` 实体，并支持本地/远程状态同步。
- 🧠 **LLM 灌溉决策**：大模型实时读取 `土壤湿度`、`未来24小时降雨概率`、`作物生长阶段`，输出 `灌溉建议` 并自动执行。
- 🔗 **多设备场景联动**：
  - 土壤湿度 < 阈值 **且** 未来6小时无雨 → 自动启动喷灌
  - 虫情灯触发警报 → 暂停喷灌，避免冲刷药剂
  - 光伏板发电量过低 → 降级为定时灌溉，关闭 4G 高功耗数据上报
- 💬 **自然语言交互**：“给番茄区浇10分钟水，但明天如果下雨就跳过” —— LLM 解析后修改自动化。
- 📊 **数据沉淀**：所有操作日志和传感器数据存入本地数据库，可供未来训练专用农业模型。

## 🧰 硬件依赖（本项目使用）

| 组件 | 型号/说明 | 接入方式 |
|------|-----------|----------|
| 商业智能喷灌控制器 | `SmartIrrigator Pro` (太阳能+4G) | 逆向云 API → HA 自定义集成 |
| 土壤湿度传感器 | Xiaomi Mi Flora / 自制电容式 | BLE → ESPHome → MQTT → HA |
| 微型气象站 | WS-2902 或 OpenWeather API | RESTful → HA 集成 |
| 边缘计算设备 | Raspberry Pi 4 / 工控机 | 运行 HA + Ollama (LLM) |
